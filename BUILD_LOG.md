# LOOP Build Log

This document tracks our progress through the strict, step-gated build phases of the LOOP AI Customer-Feedback Intelligence Platform.

## Phase 0: Kickoff & Environment

- **Step 0.1 - Confirm Scope and Stack**: Confirmed the Next.js 14 stack, four AI features, and four milestones. Verified presence of PostgreSQL connection string, Anthropic API key, and Vercel account.
- **Step 0.2 - Scaffold the App**: Created Next.js + TypeScript + Tailwind app, installed dependencies (`prisma`, `next-auth`, `@anthropic-ai/sdk`, etc.), created `.env.example` and `.gitignore`. Successfully deployed to Vercel and proved CI/CD pipeline.
- **Step 0.3 - Establish Project Conventions**: Added ESLint + Prettier config, established the `src/lib` folder skeleton for the OOP service layer (`services/`, `repositories/`, `interfaces/`, `container.ts`), and created this `BUILD_LOG.md`. (Includes bug fix for ESLint/Prettier on `fix/p0-3-eslint-config`).
- **Step 0.4 - Establish Git Workflow**: Confirmed origin remote and `main` branch. Demonstrated the step-gated git workflow (branch, trivial change, commit, push, wait for approval, merge).

## Phase 1: Foundation & Data Layer

- **Step 1.1 - Prisma Schema**: Designed and migrated the Prisma schema for Workspace, User, Feedback, Theme, FeedbackTheme, Embedding, and Report, strictly adhering to `workspaceId` tenant scoping. Configured Supabase pooled and direct connection strings.
- **Step 1.2 - Seed Script**: Built `prisma/seed.ts` utilizing `bcryptjs` for secure password hashing. Successfully seeded a demo workspace with 3 users (ADMIN, ANALYST, VIEWER), 8 themes, and 120 feedback items logically distributed and historically dated.
- **Step 1.3 - Authentication (NextAuth)**: Configured NextAuth Credentials provider using a stateless JWT session strategy. Implemented strict server-side Zod validation and transactional Workspace + User creation via `AuthService`. Attached `role` and `workspaceId` to the session JWT for multi-tenant RBAC. Secured routes using Next.js middleware and implemented cinematic UI for `/login` and `/signup`.
- **Step 1.4 - Workspaces + RBAC Foundation**: Built `TenantScopedRepository` base class guaranteeing workspace isolation across domain repos. Implemented `AuthorizationService` for strict server-side role enforcement (returning 403 Forbidden). Built the admin-only member invite flow (`AuthService.inviteMember`) and `/settings/members` dashboard view.
- **Step 1.5 - Basic Feedback CRUD (Milestone 1)**: Built Feedback CRUD, enforced channels via Zod enum, restricted POST to ADMIN/ANALYST via AuthorizationService, implemented cinematic Inbox UI, and added a reusable ConfirmModal for logout. Milestone 1 verified.

## Phase 2: Core Data Workflows

- **Step 2.1 - CSV Bulk Upload**: Implemented bulk upload via `papaparse` on the client, validated on the server with Zod. Strictly mapped headers, enforced a `<5MB` / `<5,000` rows limit, injected `workspaceId` server-side, and built a cinematic drag-and-drop modal with template generation and per-row error reporting. Made `prisma/seed.ts` idempotent.
- **Step 2.2 - Channel Ingestion (Simulation)**: Implemented simulated integration ingestion. Created `POST /api/integrations/simulate` strictly scoped to ADMIN/ANALYST roles. Built a custom `FeedbackGenerator` utilizing sentence fragments and a `Set` to prevent duplicate generation. Added a sleek "Simulate Channel" dropdown to the Inbox UI, hidden from viewers.
- **Step 2.3 - Feedback Inbox**: Upgraded the static inbox to a dynamic triage tool. Implemented server-side, offset-based pagination and full-text search clamped by strict server limits. Added an optimistic UI for inline status updates (`NEW` → `REVIEWED` → `ACTIONED`) that falls back gracefully on error. Replaced native browser alerts with a globally available, animated custom Toast system.
- **Step 2.4 - Dashboard Charts (Milestone 2)**: Built the dashboard visual centerpiece. Implemented Recharts-based volume, sentiment, and theme charts, alongside three stat cards. Designed a robust empty state for new workspaces. Set up the API structure to return a mix of real data (using Prisma aggregations for volume/counts) and mock data (for AI-dependent sentiment/themes), ensuring a zero-rewrite swap in Phase 3.

## Phase 3: AI Integration

- **Step 3.1 - AI Service Wiring**: Designed the generic `IAIProvider` interface wrapping all AI operations (`generateText`, `generateStructured`, `generateEmbedding`) to decouple domain logic from vendor specifics. Implemented `AnthropicAIProvider` and `GoogleAIProvider` (via `@google/generative-ai` with `gemini-3.1-flash-lite`). Configured `GoogleAIProvider` as the permanent primary backend to utilize generous free-tier rate limits, ensuring stable testing and backfilling without hitting quota walls. Ensured strict security by enforcing Next.js `server-only` execution on all providers.
- **Step 3.2 - Auto-classification (AI1)**: Integrated the classification engine. Added `ClassificationService` to automatically analyze incoming feedback via the active AI provider, returning strict Zod-validated JSON containing sentiment, score, feature area, rationale, and themes. Wired this into both single-item creation and a robust background batch endpoint (`/api/feedback/classify-batch`) with provider-level rate limiting, a 3-consecutive-failure circuit breaker, and defensive snake_case normalization. UI now displays rich AI metadata in the inbox and supports on-demand re-classification.
- **Step 3.3 - Theme Clustering & Trends (AI2)**: Replaced mock dashboard data with live SQL/Prisma aggregations. Built `TrendAnalysisService` for time-series aggregation to detect surging themes across trailing 7-day windows, correctly flagging new themes to prevent divide-by-zero errors. Upgraded Dashboard UI with a "Trending This Week" panel and added theme-based click-to-filter capability across the inbox, securely scoped to the current tenant.
- **Step 3.4 - Ask LOOP (AI3)**: Built a retrieval-augmented generation (RAG) feature allowing natural language querying over the feedback dataset. Upgraded the Prisma schema (handling vector dimension fixes via manual migration) and created an `EmbeddingRepository` utilizing pgvector's cosine distance operator. Implemented `AskLoopService` with strict prompt grounding to ensure zero hallucination, correctly refusing irrelevant questions. Built a dedicated `/dashboard/ask` conversational UI accessible to all roles. Added an ingestion hook for embedding new feedback and a quota-aware `POST /api/feedback/embed-batch` endpoint for backfilling history. **Note**: Embedding backfill utilizes the same chunked, rate-limited pattern as classification and requires manual repeated clicks to process large backlogs.

## Phase 4: Reporting & Export

- **Step 4.1 - Voice-of-Customer Report (AI4)**: Built the executive reporting engine. Pre-computed precise stats (sentiment counts, top themes) and verified verbatim quotes directly via `VocReportService` before passing them to the AI, ensuring complete grounding and mathematically impossible hallucinations. Added a sparse-data guard that blocks report generation if fewer than 5 feedback items exist in the requested period. Built `/dashboard/reports` UI for ADMIN/ANALYST users to generate reports across configurable date ranges. Integrated `@tailwindcss/typography` and `react-markdown` for beautiful cinematic rendering. Implemented a PDF export flow utilizing specialized `@media print` CSS rules to strip app chrome and adjust coloring for print, allowing instant professional output.
- **Step 4.2 - Export & Sharing**: Folded into Step 4.1, no separate work needed.
- **Step 4.3 - Hardening & Polish**: Executed final quality pass. Implemented strict mobile responsiveness supporting widths down to 320px without layout overflow. Added a cinematic `not-found.tsx` and an intelligent global `error.tsx` distinguishing between standard crashes and `403` permissions errors. Hardened Ask LOOP RAG feature with an empty-state check, preventing expensive AI calls when `0` embeddings exist and gracefully alerting the user. Delivered a comprehensive accessibility pass, ensuring modals correctly trap focus and respond to Escape key presses. Build verified.
- **Step 4.4 - Cinematic Design Pass**: Revamped the UI into a premium SaaS product. Replaced the generic Next.js landing page with a cinematic dark-mode hero featuring a 3D parallax tilt effect (built with Framer Motion and CSS 3D transforms to avoid the bundle/performance overhead of WebGL/react-three-fiber, while respecting OS-level `prefers-reduced-motion` settings). Upgraded the dashboard with Framer Motion `staggerChildren` entrance animations and an `AnimatedCounter` component for ticking stat cards. Enhanced auth screens with a shared glowing animated mesh and slide-up card entries, unifying the design language across the application using deep indigo and purple accents.
